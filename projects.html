Loneliness in the UK w.r.t. movement for education ( github) (2020) [Python, sklearn, Jupyter, (geo)pandas]

An investigation to find a relation between loneliness and movement for education within the UK. I tried to use previous work that related the quality of education in a given area to a loneliness index's value. My idea was that local competitiveness in term of educations might be a better indicator of movement for education than the quality of education. I created a metric in which the index for quality of education is divided by the maximum adversary in a given radius.
Involved a lot of data preprocessing using pandas and geopandas, as well as basic linear regression using sklearn and statsmodel (for residual analysis). The main thing was to find the most relevant available data and analyse the result. All of it is recorded in Jupyter notebooks found in the github link. A more complete report/explanation is coming soon.

Body/Face detection in images (2019) [Python, sklearn, skimage]

Trained two machine learning algorithms to detect faces and bodies using the corresponding data sets. The detection was made using a sliding window technique and classification through histograms of gradients and svm.
In order to prevent a potentially large number of false positives the training was done twice: first with a classification training set, and then with the same training set in addition to false positives encoutered in our detection test sets.

Data warehousing projects

Election results (2018) [Python, cassandra]

Designed a documented database storing election results from an existing csv file. We used the dimensional approach of data warehousing, by specifying dimensions and their granularity in a UML model. Implemented it in a non-relationnal and column-oriented database using cassandra. Chose a partition key to make the database horizontally scalable while still having reasonable partition sizes. Materialized views of the database in the form of key indicators computed using MapReduce operations. Explained all of our choices in a technical report. Link the report (in french).

Documentary database of articles w/ search engine (2018) [PERL, Python, Java, Antlr, XML, SQL]

Created a database of articles from various issues of a magazine. Implementated a search engine. Several step process :
● Captured the structure of HTML articles into organized XML files containing their page, magazine number, author, and so on. Used PERL scripts and RegEx to retrieve relevant information within the web pages as well as tool for verifying that the XML files produced were complete and valid.
● Transformed the text of article titles and context to keep only the most importants words according to the tf-idf coefficient. Transformed those words into lemmas according to a truncature algorithms (removed suffixes). Created index files in which lemmas where associated to articles in which they appeared.
● Creating a method that finds the corresponding lemma in a research query by using the number of common letters or Levenshtein distance.
● Computed SQL queries corresponding to natural language queries such as "give me all the articles talking about X before 2010" using AntlrWorks grammars.
● Used a Java programm to create a search engine that transformed queries into SQL queries and gave the results.

Misc. computer science projects

Numerical experiments for a research project (2019) [C++, GSL]

Over the course of my 2019 research internship in Heudiasyc Laboratory (CNRS), I studied strategies of questionning to elicit a single's users preferences regarding many competing alternatives. More specifically, I extended a well-known strategy in a framework that could account for uncertainty in the user's answer. More details in the corresponding paper : [ Preference Elicitation with Uncertainty: Extending Regret Based Methods through Belief Functions, P.-L. Guillot and S. Destercke (2019)]. In order to validate the strategy several numerical experiments were made. The idea was first to prove that it is actually different than the original one in practice, measure how conservative it is, and get a sense of how good it is at detecting inconsistency.
It was important for me to have a clean implementation for those simulations because the research project is meant to be extended to other elicitation strategies and I wanted to make something that could be easily adapted. I decided to only use C++ and GNU's Scientific Library to optimize execution time and create a good OOP architecture for the complex mechanisms behind the simulation.
There is a github project for this but unfortunately it has to be private.

Complete object oriented calculator w/ GUI ( github) (2016) [C++, Qt]

This projet was a way to practice C++ and OOP. It consists in a Reverse Polish Notation calculator with the following features :
● A stack of litterals on which to act with operators, dynamically managed so that you can use UNDO et REDO
● A way to write expressions in their regular form (not reverse polish) and evaluate them
● A way to write and save programs (list of numbers and operators) and variables
● A way to save them in text files and load theme
● A Qt interface
Its structure is so that is it easy to add other features, and more specifically different operators. Some finishing touches are still required. The Qt interface is incomplete and there are no exception handling. It was however very interesting to conceptualize and develop.
